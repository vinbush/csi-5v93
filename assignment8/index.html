Single-switch auto-scanning calculator

Press Enter to simulate the switch being pressed. Scan speed is 500 ms (1/2 second). View alternate layout.

Trial: Alternate calculator layout

Hypothesis: An alternate calculator layout (see link above) that separates the numbers from the operations will improve single-switch performance.

Rationale: The traditional calculator layout is not designed for row/col scanning, and it mixes numbers and operations in the same rows. A layout that separates out the numbers and operations may improve performance.

Test procedure

The following series of equations were input:

  1. 936 / 18 * 271 + 2961
  2. 299 / 13 * 12 - 8294
  3. 297 / 11 * 967 * 95
  4. 246 * 64 + 9526 + 453
  5. 743 * 74 - 14 - 2230

Questions generated with the help of mathsbot.com, and rearranged to simplify order of operations.

Each keyboard layout was used to input the five equations three times each. The total time to complete the set and the number of errors were recorded. I considered an error to be when I pressed an incorrect number or operation (so pressing the wrong row and letting it run to the end and go back to row scanning was not considered an error).

Results

Original Layout:

  1. 4:40, 0 mistakes
    2. 280
  2. 4:40, 1 mistake
    3. 280
  3. 5:03, 1 mistake
    4. 303

Alternate Layout:

  1. 4:48, 0 mistakes
    2. 288
  2. 5:14, 1 mistake
    3. 314
  3. 4:20, 0 mistakes
    4. 260

Mean time for original: 287.67. Mean mistakes for original: .67

Mean time for alternate: 287.33. Mean mistakes for alternate: .33

Discussion

There doesn't appear to be any significance between the two layouts. Anecdotally, I didn't really notice any difference between the two. While using the alternate layout, I thought it was nice to have the operations all in one place so I always knew when an alternate was coming up, but it also increased the time it took to circle back to the beginning of the numbers, so it felt like a wash.
